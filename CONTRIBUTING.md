# Contributing to RepoPush Demo

Thanks for helping improve RepoPush Demo. This contribution path is designed to keep triage predictable and low-friction.

## How to contribute

1. Search existing issues before opening a new one.
2. Use the bug, feature request, or question issue form.
3. For code changes, open a pull request with a focused scope.
4. Include tests or verification notes for behavior changes.
5. Keep launch, beta, roadmap, and deprecated claims clearly labeled.

## Pull request checklist

- [ ] The change has a clear problem statement.
- [ ] Documentation is updated when user behavior changes.
- [ ] No secrets, tokens, or private URLs are included.
- [ ] Roadmap functionality is not described as stable launch behavior.
- [ ] Security-sensitive changes reference SECURITY.md.

## Review process

Maintainers prioritize reproducible bugs, small pull requests, and documentation that helps users complete the quickstart path.
