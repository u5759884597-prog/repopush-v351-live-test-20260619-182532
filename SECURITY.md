# Security Policy

## Supported versions

Latest release candidate and active beta versions when marked beta.

## Reporting a vulnerability

Email: security@example.com

Include affected versions, reproduction steps, expected impact, and any safe proof-of-concept details. Do not include secrets or private user data in reports.

## Disclosure policy

Report privately first; maintainers will coordinate disclosure after validation.

## Scope notes

Roadmap features are not part of the supported security surface until they are released. Beta features may have additional limitations documented in ROADMAP.md.
