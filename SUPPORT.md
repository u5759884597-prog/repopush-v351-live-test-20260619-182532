# Support

## Where to get help

- Questions: open a question issue using `.github/ISSUE_TEMPLATE/question.yml`.
- Bugs: open a bug report with reproduction steps.
- Security issues: follow SECURITY.md instead of opening a public issue.
- Email: support@example.com

## Response policy

Community issues are triaged weekly; security reports follow SECURITY.md.

## Troubleshooting checklist

1. Confirm your version and installation method.
2. Re-run the quickstart from docs/QUICKSTART.md.
3. Search existing issues.
4. Open a question issue with logs that do not contain secrets.
